PHP responds differently to HTTP requests based on the type of request sent (GET, POST, PUT, PATCH, DELETE, etc.) as well as the format in which the data is sent:

  • JSON: "application/json"
  • FormData: "multipart/form-data"
  • Query String: "application/x-www-form-urlencoded"

Consequently, receiving data in PHP has to be handled differently in each case.

For POSTs, JSON must be handled differently than FormData and query strings. For PUT, PATCH, and DELETE requests, each data format must be handled uniquely by the PHP script. Ultimately, it's easier to just use POST requests for updating and deleting data, but it's still good to know how to respond to all possible scenarios.

Note: If possible, using JSON is ideal for all request types. The benefit of JSON is that it doesn't turn all values into strings. Yes, it's stringified/parsed (encoded/decoded), but once that's done the value types are intact. The other formats just turn everything into a string.
